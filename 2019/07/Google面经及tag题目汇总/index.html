Shortest Unsorted Continuous Subarray

581. Shortest Unsorted Continuous Subarray

思路

这是一道easy的题目，最暴力的解法是直接sort然后对比，把不一样的一段截取出来。但$O(n)$的解法却委实很难想到。这里用到了双向双指针。

我们要想一个问题，这个要求的子数组满足什么样的情况。

  1. 作为开头的这个数字的前一个数字，小于之后所有的数字。
  2. 作为结尾的这个数字的后一个数字，大于之前的所有数字。

因此，我们设置一个int max等于第一个数字从前向后遍历，如果当前的数字大于max的话我们就更新max,符合当前数字大于之前所有数字我们不做任何动作，如果当前数字小于max的话，说明出现了逆序，我们把当前遍历到的index更新为end。

同理，设置一个int min等于最后一个数字从后相前遍历，如果当前的数字小于min的话我们就更新min,符合当前数字小于之后所有数字我们不做任何动作，如果当前数字大于min的话，说明出现了逆序，我们当前遍历到的index更新为start。

这一题还需要注意的是初始化 start = -1, end = -2因为如果数组从头到尾都是ascending的话，end-start+1等于0。

时间复杂度

time ：$O(n)$

space : $O(1)$

560. Subarray Sum Equals K

思路

这是一道子数组题，所用的方式和前四百里的325. Maximum Size Subarray Sum Equals k一样。

遍历数组，用一个HashMap存储直至当前位置的sum和这个sum值出现过几次的映射关系（因为可以存在负数，所以同一个sum可能反复出现），然后通过在map里查找$sum-k$累加res得到结果。

时间复杂度

Time ： $O(n)$

Student Attendance Record II

552. Student Attendance Record II

这是551. Student Attendance Record I的follow up，但511实在太简单了就不说了。

思路

因为题目的原意是，至多只能有一个A，如果超出一个则不符合，所以我们一开始不考虑有A的String，我们只需要先求出不含有A的String的排列组合个数，然后再计算每个位置替换成A的个数就可以了。

那么问题就变成了，我们如何计算不含有A的全部排列组合个数？

我们考虑字符串以PorL结尾。

我们考虑 $i$ 个字母组成的String，这个String分别以P或者L结尾，我们用一个数组PorL[i]来存储这个结果。

那么PorL的递推公式应该是什么呢？先来想一个问题，PorL[i]应该由什么组成？自然是P[i] + L[i]那么P[i] + L[i]又该由什么组成？如上图所示:

下面就是要求包含一个A的情况，那么我们就得去除一个字符，从而给A留出位置。那么就相当于在数组的任意一个位置上加上A，那么数组就被分成左右两个部分了，而这两个部分当然就不能再有A了，所有不包含A的情况都已经在数组PorL中计算过了，而分成的子数组的长度又不会大于原数组的长度，所以我们直接在PorL中取值就行了，两个子数组的排列个数相乘，然后再把所有分割的情况累加起来就是最终结果。

注意M要写成$M = 1000000007$, 符号^是异或的意思。
